# HTML Presentation Workflow For KIT

Use this workflow for standalone HTML presentations and commercial proposals.

## Purpose

Create presentation artifacts for KIT without adding them to the production website and without introducing build dependencies.

## Rules

Presentations should be:
- a single standalone HTML file;
- inline CSS;
- no npm requirement;
- no external dependencies unless explicitly approved;
- mobile-readable;
- print-friendly when useful;
- visually aligned with KIT style.

Do not:
- integrate the presentation into the production site;
- add fake reviews, projects, ratings, counters, or claims;
- imply exact final price online;
- use production secrets or real lead PII.

## Recommended Structure

1. Cover.
2. Client need or problem.
3. KIT offer.
4. Process.
5. Materials/options.
6. Price factors.
7. Honest trust/conditions.
8. Next step / CTA.

## Verification

Check:
- open the HTML locally;
- desktop layout;
- mobile width;
- print view if relevant;
- no external dependencies unless approved;
- no fake claims.

## Output

Return:
- summary;
- file path;
- verification performed;
- notes and limitations.
