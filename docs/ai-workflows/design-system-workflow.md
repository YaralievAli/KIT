# Design System Workflow

Use this workflow when creating or reviewing KIT UI, landing pages, presentations, or visual docs.

## Source Of Truth

Before visual work, read:
- `AGENTS.md`;
- `docs/design/DESIGN.md`;
- relevant component/page files;
- the user's screenshots or visual notes.

## Design Direction

Preserve:
- premium dark teal / graphite / champagne palette;
- warm kitchen photography;
- clear mobile layout;
- readable Russian text;
- practical commercial sections;
- honest trust signals.

Avoid:
- generic SaaS visuals;
- purple/blue gradient-heavy styling;
- tiny text;
- over-dense mobile cards;
- fake ratings/reviews/projects;
- unsupported luxury or production claims.

## Page Quality Checks

For pages and sections:
- H1 is clear and not keyword-stuffed;
- CTA matches intent;
- sections are scannable;
- cards do not become nested or visually heavy;
- mobile has no horizontal overflow;
- text does not overlap or clip;
- important links and buttons are visibly interactive.

## SEO Landing Pages

SEO pages should feel like commercial landing pages, not article dumps.

Prefer:
- hero with clear intent;
- practical explanation;
- pros/limitations;
- price factors;
- materials/options;
- process;
- related links;
- visible FAQ;
- CTA to calculator/contact.

Structured data must match visible content.

## Review Output

Return:
- visual verdict;
- desktop/mobile findings;
- screenshots path if used;
- issues by severity;
- exact implementation recommendations.
