# KIT HTML Presentation Brief

Use this brief to create a standalone HTML presentation or commercial proposal for KIT.

## Goal

Create a polished single-file HTML presentation for:
- client commercial proposal;
- company introduction;
- materials/fittings explanation;
- process overview;
- project estimate explanation;
- internal sales deck.

## Output Requirements

- Single `.html` file.
- Inline CSS.
- No npm/build step.
- No external dependencies unless explicitly approved.
- Mobile-readable.
- Print-friendly if useful.
- Not integrated into the production website.

## Visual Style

Use KIT visual language:
- premium dark teal;
- graphite/dark neutral backgrounds;
- champagne/gold accents;
- warm kitchen imagery if available;
- rounded cards;
- clear CTA;
- readable Russian text.

Avoid:
- generic SaaS gradients;
- fake luxury;
- tiny text;
- clutter;
- fake review/rating/project elements.

## Suggested Structure

1. Cover:
   - KIT name;
   - clear offer;
   - service area: Saint Petersburg and Leningrad Region.
2. Client need:
   - space, storage, materials, budget, timing.
3. KIT offer:
   - measurement/consultation;
   - planning;
   - kitchen production/order flow;
   - installation if true for the business process.
4. Materials and options:
   - facades;
   - countertops;
   - fittings;
   - lighting;
   - storage systems.
5. Process:
   - request;
   - consultation;
   - measurement/project;
   - estimate;
   - contract;
   - production/order;
   - installation.
6. Price factors:
   - size/layout;
   - materials;
   - fittings;
   - countertop;
   - appliances integration;
   - installation complexity.
7. Honest conditions:
   - online estimate is preliminary;
   - final price depends on measurement, project, materials, and selected options.
8. Next step:
   - clear CTA to call, Telegram, calculator, or consultation.

## Claims Rules

Do not include:
- fake reviews;
- fake ratings;
- fake completed projects;
- fake counters;
- unsupported guarantees;
- exact final online price;
- exact production deadlines unless approved;
- unsupported legal or warranty claims.

## Verification

Before delivery:
- open locally;
- check desktop;
- check mobile width;
- check print if relevant;
- verify no external dependencies unless approved;
- verify no fake claims or real PII.
